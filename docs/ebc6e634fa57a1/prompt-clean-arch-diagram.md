# nanobanana 用プロンプト: クリーンアーキテクチャ同心円図

## プロンプト案 1（シンプル）

```
クリーンアーキテクチャの同心円図を描いてください。

4層構成で、外側から内側に向かって：
1. Frameworks & Drivers（最外側・薄い青）
2. Interface Adapters（青）
3. Use Cases（黄色）
4. Entities（中心・オレンジ）

矢印は外側から内側に向かって「依存方向」を示してください。
日本語ラベルは不要、英語のみで。
シンプルでモダンなデザイン、背景は白。
```

---

## プロンプト案 2（本記事の対応も入れる）

```
クリーンアーキテクチャの同心円図を描いてください。

4層構成で、外側から内側に向かって：
1. Frameworks & Drivers / Infrastructure（最外側・グレー）
2. Interface Adapters / Presentation（水色）
3. Use Cases / Application（黄色）
4. Entities / Domain（中心・オレンジ）

各層にはスラッシュで区切って2つのラベルを入れてください。
左側がクリーンアーキテクチャの用語、右側が本記事での用語です。

矢印は外側から内側に向かって依存方向を示してください。
シンプルでモダンなデザイン、背景は白。
```

---

## プロンプト案 3（日本語併記）

```
クリーンアーキテクチャの4層同心円図を作成してください。

外側から内側へ：
1. Infrastructure（技術的詳細）- グレー
2. Presentation（入出力）- 水色
3. Application（ユースケース）- 黄色
4. Domain（ビジネスルール）- オレンジ

依存の矢印は外から内へ。
各層に英語名と日本語名を併記。
シンプル、フラットデザイン、白背景。
```

---

## 備考

- 本記事では Presentation と Interface Adapters が対応、Infrastructure と Frameworks & Drivers が対応
- クリーンアーキテクチャの図と本記事の層名を対比させたい
- 依存方向（外→内）を強調したい
